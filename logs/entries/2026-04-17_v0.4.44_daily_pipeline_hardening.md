# v0.4.44 - Daily pipeline hardening

## Context

Tras revisar los artefactos mas recientes, la cadena diaria estaba quedando demasiado repartida entre varias automatizaciones independientes.

Resultado practico observado:

- `2026-04-15` dejo live, audit y comparativa completos
- `2026-04-16` solo dejo `observation_backfill`
- `2026-04-17` todavia no habia dejado artefactos nuevos

La conclusion operativa es clara: el problema no era solo del modelo, sino de la orquestacion diaria. Si una tarea corre y otra no, la foto del sistema queda incompleta aunque el codigo de cada script este sano.

## Cambios aplicados

### 1. Nuevo orquestador diario

Se añade:

- `scripts/run_daily_pipeline.py`

Funcion:

1. ejecuta `run_blind_live_validation.py`
2. ejecuta `run_observation_backfill.py`
3. ejecuta `run_blind_snapshot_resolution_audit.py`
4. ejecuta `run_cohort_overlay_diagnostics.py`
5. opcionalmente permite `health_check` con `pytest`

Cada paso:

- captura `started_at_utc` y `finished_at_utc`
- registra `exit_code`
- guarda `stdout_tail` y `stderr_tail`
- verifica artefactos esperados cuando aplica
- continua con el resto aunque haya un fallo

Salida consolidada:

- `logs/snapshots/<fecha>_daily_pipeline_report.json`

Estados posibles:

- `ok`
- `warning`
- `degraded`

### 2. Cobertura de tests

Se añade:

- `tests/unit/test_daily_pipeline.py`

Cobertura nueva:

- construccion del pipeline base
- degradacion cuando un paso termina sin artefacto esperado
- persistencia del reporte diario
- priorizacion de estados globales

## Verificacion

- `venv/bin/python -m pytest -q tests/unit/test_daily_pipeline.py`
- `venv/bin/python -m pytest -q`

## Lectura operativa

Este cambio no toca forecast ni pricing. Su valor esta en blindar el flujo diario para que la evidencia deje de depender de varias automatizaciones separadas que pueden desincronizarse.

La idea a partir de aqui es simple:

- una automatizacion principal ejecuta el pipeline entero
- el resto de tareas quedan como apoyo o watchdog, no como unica fuente de artefactos

## Siguiente paso recomendado

1. Reutilizar `Daily Live Snapshot` como `Daily Pipeline`.
2. Mantener el resto de automatizaciones como backup ligero o supervision.
3. Observar durante varios dias si desaparecen los cortes incompletos tipo `solo backfill`.
