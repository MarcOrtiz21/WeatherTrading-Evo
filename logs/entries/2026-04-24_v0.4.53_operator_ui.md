# v0.4.53 - Local operator UI

## Context

La consola por terminal ya centralizaba el sistema, pero seguia obligando a trabajar en texto. Se anade una UI local rapida para revisar estado, tickets y copytrading desde navegador, sin depender de frameworks ni servicios externos.

## Cambios

- Nuevo script: `scripts/run_operator_ui.py`.
- Nueva UI local en `http://127.0.0.1:8787`.
- API local:
  - `GET /api/dashboard`
  - `POST /api/export`
  - `POST /api/decision`
  - `GET /api/health`
- Nuevo test suite: `tests/unit/test_operator_ui.py`.
- Version del proyecto: `0.4.53`.

## Uso

```bash
venv/bin/python scripts/run_operator_ui.py
```

Opciones utiles:

```bash
venv/bin/python scripts/run_operator_ui.py --budget-usd 10 --max-tickets 12
venv/bin/python scripts/run_operator_ui.py --no-open
```

La UI permite:

- ver readiness/pipeline/DB/audit en una sola pantalla;
- filtrar tickets por texto y estado;
- ordenar por stake, edge o magnitud copy;
- ver copy flow por trader;
- exportar snapshot consolidado;
- registrar aprobaciones paper desde boton.

## Seguridad

La UI no envia ordenes reales ni firma payloads. El boton `Aprobar paper` solo escribe una decision local en `logs/snapshots/<fecha>_operator_decisions.jsonl`.

## Validacion

- `venv/bin/python -m pytest tests/unit/test_operator_ui.py tests/unit/test_operator_console.py -q` -> `9 passed`
- `curl -I http://127.0.0.1:8787/` -> `200 OK`
- `GET /api/dashboard` -> devuelve tickets
- `POST /api/export` -> genera `logs/snapshots/2026-04-24_operator_console.json`
