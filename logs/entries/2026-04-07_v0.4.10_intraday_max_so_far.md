# Version 0.4.10

## Contexto

Esta iteracion abre la puerta a operativa intradia en mercados del mismo dia. El objetivo es dejar de tratar todos los mercados como puro forecast diario cuando, en plazas como Shanghai al final de la tarde local, ya existe una maxima realizada o casi realizada.

## Decisiones tecnicas

- Se crea un contexto reusable `intraday max-so-far`.
- Para mercados del mismo dia (`horizon_days = 0`), el baseline puede construir una distribucion distinta:
  - maximo ya observado hasta el momento
  - horas restantes del dia
  - menor dispersion que un forecast diario completo
- El contexto intradia intenta primero leer observaciones locales desde `weather_trading.db`.
- Si no hay observaciones locales, hace fallback a un proxy usando la trayectoria horaria de Open-Meteo.
- Se anade `ZSPD` al catalogo para que Shanghai deje de caer por `missing_station_catalog`.
- La validacion live ya guarda metadata intradia en el snapshot:
  - `intraday_active`
  - `intraday_source`
  - `intraday_max_so_far_c`
  - `intraday_remaining_hours`

## Resultado esperado en mercado tardio

- Un mercado del mismo dia puede seguir abierto formalmente aunque la maxima local ya se haya producido.
- Con esta mejora, el sistema ya puede distinguir mejor entre:
  - dia apenas empezado
  - tarde con muchas horas ya realizadas
  - cierre practicamente decidido

## Riesgos o supuestos

- El fallback intradia por forecast horario no es equivalente a una observacion contractual real.
- Para mercados resueltos por Wunderground u otra fuente contractual, la mejor version de esta logica seguira siendo la que use observaciones locales reales o la fuente contractual directamente.
- La mejora es operativamente util, pero aun no convierte por si sola el sistema en ejecutor intradia perfecto.
