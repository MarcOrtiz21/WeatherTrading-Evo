# Version 0.4.7

## Contexto

Esta iteracion prepara el proyecto para moverlo con menos friccion desde macOS a Windows y continuar el trabajo desde otra maquina sin depender del entorno virtual local.

## Archivos creados o modificados

- `README.md`
- `docs/windows_setup.md`
- `scripts/bootstrap_windows.ps1`
- `scripts/run_adaptive_policy_stability.py`
- `scripts/run_multidate_policy_stability.py`
- `scripts/run_blind_snapshot_resolution_audit.py`
- `src/weather_trading/__init__.py`
- `pyproject.toml`

## Decisiones tecnicas

- Se documenta una instalacion limpia en Windows basada en `pyproject.toml`.
- Se anade un bootstrap en PowerShell para crear `venv`, instalar dependencias y lanzar la suite.
- Las nuevas corridas de estabilidad y auditoria ya guardan rutas relativas en snapshots en vez de paths absolutos del sistema local.
- La documentacion nueva evita depender de `venv/bin/python`, que era una convencion de macOS/Linux.

## Resultados validados

- `venv/bin/python -m pytest -q`
- Resultado: `66 passed`

## Riesgos o supuestos

- Los snapshots historicos ya generados antes de esta version siguen conteniendo rutas absolutas de macOS.
- Eso no bloquea la ejecucion en Windows, pero deja un rastro historico menos portable en esos archivos antiguos.
- El script `bootstrap_windows.ps1` no se ha ejecutado en esta maquina porque el entorno actual es macOS; queda preparado para uso en PowerShell.

## Siguiente paso recomendado

1. Clonar o copiar el repo en Windows.
2. Ejecutar `.\scripts\bootstrap_windows.ps1`.
3. Confirmar con `python -m pytest -q`.
4. Lanzar una validacion live corta y comprobar que los snapshots nuevos quedan con rutas relativas.
