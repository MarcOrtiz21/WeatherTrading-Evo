# v0.4.58 - Launchd relocation and resource limits

## Context

The daily macOS LaunchAgents were installed but could not execute the project from `~/Documents`, failing with exit code 127 before opening the runner script. After moving the project outside the macOS protected Documents tree, the runner could execute, but the full daily job exposed a second launchd-only issue: the audit step could hit SQLite/zoneinfo resource errors under the lower default file descriptor limit.

## Changes

- Moved the working repo to `/Users/marc/Developer/WeatherTrading-Evo`.
- Left `/Users/marc/Documents/Polymarket/WeatherTrading` as a symlink to the new location for compatibility.
- Reinstalled both LaunchAgents so their `ProgramArguments`, `WorkingDirectory`, stdout, and stderr paths point at `/Users/marc/Developer/WeatherTrading-Evo`.
- Added `SoftResourceLimits` and `HardResourceLimits` with `NumberOfFiles=4096` to generated LaunchAgent plists.
- Added a `ulimit -n 4096` guard and open-file-limit logging to the launchd runner.
- Made blind snapshot resolution audit local DB reads use SQLite read-only URI mode.
- Added a fallback path so local SQLite open failures do not crash the audit; the resolver falls back to remote archive resolution when allowed.
- Added tests for the audit fallback and launchd resource limits.

## Verification

- `launchctl kickstart` daily pipeline: exit code 0.
- `launchctl kickstart` watchdog: exit code 0.
- Operator console now reports both LaunchAgents as `ok`.
- Focused tests: `35 passed`.
