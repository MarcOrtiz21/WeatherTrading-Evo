# v0.4.45 - Live snapshot idempotent reruns

## Context

Tras endurecer el pipeline diario, el primer rerun real sobre `2026-04-15` detecto un falso fallo operativo:

- `blind_live_validation` salia con `exit_code=1`
- pero el snapshot `2026-04-15_polymarket_blind_live_validation.json` ya existia
- el resto del pipeline seguia sano

La causa era simple: el script live siempre intentaba redescubrir mercados y tocar red aunque el snapshot del dia ya estuviera congelado.

## Cambios aplicados

- `scripts/run_blind_live_validation.py`
  - se añade `--force-refresh`
  - por defecto, si el snapshot del dia ya existe, se reutiliza y se imprime su resumen
  - se extrae `build_snapshot_path`
  - se extrae `print_snapshot_summary`
- test nuevo:
  - `tests/unit/test_blind_live_validation_script.py`

## Efecto operativo

La validacion live pasa a ser idempotente para reruns del mismo dia:

- si ya existe snapshot valido, se revisa ese artefacto
- no se vuelve a depender de red innecesariamente
- el pipeline diario deja de contar ese caso como fallo real

`--force-refresh` queda disponible cuando queramos forzar una corrida nueva a pesar de existir snapshot.

## Verificacion

- `venv/bin/python -m pytest -q tests/unit/test_blind_live_validation_script.py tests/unit/test_daily_pipeline.py`
- `venv/bin/python -m pytest -q`
- `venv/bin/python scripts/run_daily_pipeline.py --reference-date 2026-04-15`

## Conclusion

Este ajuste no mejora el forecast, pero si mejora bastante la disciplina diaria:

- menos falsos negativos operativos
- mas reruns seguros
- y mejor separacion entre fallo de red real y snapshot ya congelado
