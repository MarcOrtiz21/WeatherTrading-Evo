# Version 0.4.6

## Contexto

Esta iteracion aplica la primera mejora fuerte sobre precision de forecast: el baseline deja de construir la distribucion de maxima diaria desde un `daily max` puntual y pasa a usar la trayectoria horaria completa cuando esta disponible.

## Archivos creados o modificados

- `src/weather_trading/services/forecast_engine/openmeteo_distribution.py`
- `scripts/run_blind_live_validation.py`
- `tests/unit/test_openmeteo_distribution.py`
- `src/weather_trading/__init__.py`
- `pyproject.toml`

## Decisiones tecnicas

- El builder base ahora tiene dos modos:
  - `hourly_path` cuando existen temperaturas horarias
  - fallback gaussiano cuando no existen
- En modo `hourly_path`:
  - se simula la maxima diaria desde la curva horaria
  - se combina sesgo global y ruido horario correlacionado
  - la salida queda discretizada por bins enteros de temperatura
  - la simulacion es determinista para el mismo input
- La validacion live ya alimenta el baseline con:
  - `model_hourly_temps`
  - `model_cloud_cover_avg`

Lectura:

- El baseline horario comprime menos la distribucion y genera mucha mas conviccion local en bins concretos.
- Eso aumenta claramente el edge aparente y el numero de trades paper posibles.
- Pero no demuestra por si solo mas precision final: de momento es una mejora estructural prometedora, no una victoria validada ex post.

## Riesgos o supuestos

- El nuevo baseline puede estar sobreconcentrando masa en torno a ciertos bins si el ruido simulado es demasiado optimista.
- La metrica `market mode match` en live no es una medida final de acierto; solo da una señal rapida de alineacion con el mercado.
- La prueba real de esta mejora llegara cuando el auditor ex post empiece a resolver snapshots ya maduros.

## Siguiente paso recomendado

1. Ejecutar el auditor ex post desde el `7 de abril de 2026` para medir si este baseline horario mejora `log loss`, `Brier` y `paper PnL`.
2. Si la mejora es real, extender la misma idea al camino `optimized` y no solo al `baseline`.
3. Si el edge aparente sube pero el `paper PnL` no acompaña, recalibrar la dispersion del simulador horario antes de seguir añadiendo complejidad.
