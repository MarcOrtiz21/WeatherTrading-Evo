# WeatherTrading v0.4.55 - Operator health and log-loss guardrails

## Context

After reviewing the daily automations, the manual `2026-04-26` pipeline run completed successfully while the scheduled artifacts for `2026-04-25` were absent. This points to an operational scheduling/freshness gap rather than a broken model pipeline.

## Changes

- Added `system_health` to the operator dashboard payload with latest successful pipeline date, recent missing daily reports, automation status, and a concrete recovery command.
- Added a local UI `Recover` action backed by `/api/recover`, which runs `scripts/run_daily_pipeline.py` for the selected reference date and returns the refreshed dashboard.
- Added preflight warnings when model log loss materially underperforms market log loss.
- Added ticket-level `risk_controls` with explicit stake multipliers and notes.
- Reduced supervised stake sizing for Fahrenheit range bins, other Fahrenheit families, extreme fair probabilities, and tail prices.
- Preserved the existing opposed-watchlist veto and surfaced copytrading conflicts as risk notes.

## Result

The `2026-04-26` operator console now reports:

- Pipeline `ok`
- Readiness `ready`
- Preflight `warning` but approval still allowed
- Missing recent pipeline date: `2026-04-25`
- Log-loss warning: model `1.865` vs market `1.284`
- Reduced stake suggestions on Fahrenheit/range-bin and overconfident tickets
