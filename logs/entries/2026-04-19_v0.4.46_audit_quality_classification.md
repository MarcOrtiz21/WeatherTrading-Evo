# v0.4.46 - Audit quality classification

## Context

Las ultimas auditorias dejaron una ambiguedad operativa importante:

- el snapshot `2026-04-18` conservaba metricas agregadas utiles
- pero solo sobre `160/200` eventos maduros
- y sin una marca explicita podia leerse demasiado cerca de una auditoria completa

La necesidad era endurecer el sistema para distinguir automaticamente entre:

- audit completo
- audit parcial
- audit no concluyente

## Cambios aplicados

### 1. Umbrales configurables en settings

Se añaden en `config/settings.yaml`:

- `audit_quality.min_mature_coverage_for_complete = 0.95`
- `audit_quality.min_mature_coverage_for_actionable = 0.85`

### 2. Clasificacion explicita en la auditoria

`scripts/run_blind_snapshot_resolution_audit.py` ahora calcula y persiste:

- `audit_quality.classification`
  - `complete`
  - `partial`
  - `non_conclusive`
- `audit_quality.reason`
- `audit_quality.is_actionable`
- `audit_quality.is_comparable_to_full_coverage`

La auditoria tambien lo imprime en consola junto a la cobertura madura.

### 3. Integracion con el pipeline diario

`scripts/run_daily_pipeline.py` ahora lee `audit_quality` del snapshot de auditoria:

- si el audit es `partial`, el paso queda en `warning`
- si el audit es `non_conclusive`, el paso queda en `degraded`

Asi el pipeline diario ya no depende de que alguien lea el JSON a mano para entender si el dia es comparable o no.

## Verificacion

- `venv/bin/python -m pytest -q tests/unit/test_blind_snapshot_resolution_audit.py tests/unit/test_daily_pipeline.py`
- `venv/bin/python -m pytest -q`
- `venv/bin/python scripts/run_blind_snapshot_resolution_audit.py --reference-date 2026-04-18`
- `venv/bin/python scripts/run_daily_pipeline.py --reference-date 2026-04-18`

## Resultado esperado

Con esta capa:

- un audit con cobertura madura >= `95%` se trata como `complete`
- entre `85%` y `95%` queda como `partial`
- por debajo de `85%` queda como `non_conclusive`

El corte `2026-04-18`, con `80%` de cobertura, pasa a quedar marcado formalmente como no concluyente en vez de parecer una auditoria comparable a las completas.

## Conclusion

La mejora no cambia el scoring, pero si mejora bastante la disciplina interpretativa:

- menos riesgo de sacar conclusiones fuertes sobre dias parciales
- mejor lectura para el watchdog diario
- y separacion mucho mas limpia entre degradacion operacional y degradacion real del modelo
